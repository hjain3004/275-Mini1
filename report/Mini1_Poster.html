Cumulative Speedup vs. AoS Serial

Key Optimizations

Structure of Arrays: Changed memory layout from AoS (stride-dependent) to SoA (column-oriented) to maximize cache-line efficiency.
Flat String Buffers: Eliminated ~265.2M scattered heap allocations by using a single contiguous `char[]` buffer per column, destroying string overhead.
OpenMP Parallelism: Scatter-gather query engine with `#pragma omp parallel` achieving near-linear scaling up to hardware bandwidth limits.
String Interning: Converted highly-duplicated string fields (59 unique types) to `uint16_t` enum dictionaries.
RFC-4180 Parser: Zero-copy chunking using 14-thread processing via line boundary detection in raw CSV bytes.

AoS vs. SoA Query Performance

SoA yields 1.8×–3.4× speedups directly via sequential hardware prefetching alone.

Thread Scaling Degradation (Amdahl's Law)

Performance collapses beyond 8 threads due to M4 Pro memory bandwidth saturation (~100 GB/s) and efficiency-core contention.

Multiplicative Velocity (SoA + OpenMP)

SoA and OpenMP compound formidably. Combined, they crash Date Range Query latencies from 47.6ms to 1.4ms on 5M rows from the 12 GB dataset.

Findings & Lessons Learned

Threading on Trivial Workloads: On 1,000 rows, threading is up to 8× slower. The 100K-row threshold is required to offset Thread creation and lock limits.
Memory Bottleneck: The process is inherently limited by memory I/O and RAM throughput rather than core limits.
AoS Memory Wall: AoS requires >24 GB for 20.4M rows (OOM crash). SoA flat buffers fit the entire dataset in 12.5 GB.
